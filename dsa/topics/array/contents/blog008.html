Greedy & Optimization Patterns on Arrays

Greedy algorithms feel dangerous because they make decisions you cannot undo.

This chapter teaches you:

  • When greedy is logically valid
  • Why “local best” sometimes implies “global best”
  • How to prove greedy correctness in interviews

What Does “Greedy” Actually Mean?

Greedy does not mean:

  • Fast
  • Approximate
  • Lucky

Greedy means:

At every step, make the best possible local decision that can never harm an optimal solution.

That last part is the key.

26. Greedy Choice on Arrays

The Core Question You Must Ask

If I make this choice now, can an optimal solution still exist?

If the answer is yes for all steps, greedy is safe.

The Mental Model

Think of greedy as:

  • Pruning bad futures early
  • Reducing the problem size aggressively

You are not searching all possibilities. You are ruling out impossible ones.

Why Each Line of a Greedy Solution Exists

  • Sort array → impose structure
  • Iterate linearly → process decisions in order
  • Choose locally optimal element → irreversible choice
  • Update state → reflect consequence
Sorting is often the proof step in disguise.

Why Sorting Appears So Often

Sorting converts chaos into order.

Once sorted:

  • Choices become comparable
  • Extremes dominate outcomes

Examples:

  • Minimum jumps
  • Activity selection
  • Interval scheduling

How to Start Writing a Greedy Solution

Always start with:

  • “What must an optimal solution do first?”

If the first choice is forced, greedy is implied.

27. Rearrangement Pattern

What Rearrangement Problems Are Really About

Rearrangement problems ask:

How should I permute elements to optimize a goal?

The original order is irrelevant. Only placement matters.

The Key Insight

Extreme values dominate.

Large values should often:

  • Go where they matter most
  • Avoid canceling each other

The Mental Model

Think of rearrangement as:

  • Pairing strengths with weaknesses
  • Separating conflicting elements

Why Each Line Exists

  • Sort array → rank elements
  • Place smallest/largest strategically
  • Track result → objective function
Rearrangement is greedy with spatial awareness.

Classic Rearrangement Problems

  • Maximize sum of absolute differences
  • Rearrange to avoid adjacent duplicates
  • Minimize maximum adjacent difference

How to Prove Greedy Correctness

In interviews, never say:

  • “Greedy works here”

Say:

  • “Any optimal solution can be transformed to follow my greedy choice without worsening the result”

That is the exchange argument.

Greedy vs Dynamic Programming

Greedy:

  • One path
  • No backtracking
  • Fast

DP:

  • Many paths
  • State exploration
  • Guaranteed
If greedy works, DP is overkill.

Universal Greedy Design Checklist

  1. Can I sort to impose order?
  2. Is the first decision forced?
  3. Can I prove local choice is safe?
  4. Does the remaining problem look similar?

Final Insight

Greedy is not guessing.

It is:

Logical inevitability disguised as confidence.

Once you see that, greedy problems become obvious.
