Monotonic Array Technique

This technique is what separates “good array solvers” from “elite problem solvers”.

It appears whenever a problem asks:

“For each element, find the next / previous element that is bigger or smaller.”

If you try to solve this naively, you get O(n²). Monotonic techniques reduce it to O(n).

Why Monotonic Techniques Exist

Array problems often hide this structure:

  • Every element wants to know about its neighbors
  • But only the “important” neighbors matter

The monotonic technique keeps only useful information.

What Does “Monotonic” Mean?

A structure is monotonic if:

  • It is always increasing, or
  • Always decreasing

Once this property is violated, elements are removed.

You are enforcing order dynamically.

The Core Insight

If an element is blocked by a better candidate, it will never be useful again.

Therefore:

Remove it immediately.

This single idea makes the entire technique work.

The Mental Model

Think of elements standing in a line.

  • Each new element walks in from the right
  • It knocks out weaker elements in front of it

Only the strongest survivors remain.

Why Each Line Exists (Push & Pop Logic)

  • Initialize empty structure → no assumptions
  • Iterate array once → linear pass
  • While monotonic property breaks → pop
  • Push current element → candidate for future
Each element is pushed once and popped once. That is why it is O(n).

Monotonic Increasing vs Decreasing

Choose increasing if you want:

  • Next smaller element
  • Previous smaller element

Choose decreasing if you want:

  • Next greater element
  • Previous greater element
Direction depends on the question, not the array.

How to Start Writing a Monotonic Solution

Always begin with this question:

For each index i, what am I trying to find?

Then decide:

  • Left to right or right to left?
  • Increasing or decreasing?

The rest is mechanical.

Classic Problems Solved by This Pattern

  • Next Greater Element
  • Largest Rectangle in Histogram
  • Daily Temperatures
  • Stock Span Problem

Why This Is Not Just a Stack Trick

Many people memorize this as:

  • “Use a stack”

That is wrong.

The stack is an implementation detail.

The idea is dominance elimination.

Common Mistakes

  • Choosing wrong direction
  • Using wrong comparison (< vs ≤)
  • Forgetting index vs value distinction

These are logic mistakes, not syntax mistakes.

Universal Monotonic Checklist

  1. Am I comparing neighbors?
  2. Is brute force O(n²)?
  3. Can weaker candidates be eliminated early?
  4. Do I need next/previous greater/smaller?

Final Insight

Monotonic techniques are not complicated.

They are:

A disciplined way of deleting useless information.

Once you understand that, you will see this pattern everywhere.
