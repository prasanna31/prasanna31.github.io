Task Environment Specification (PEAS)

A structured way to define what an intelligent agent should do and how its success is measured

1. What Is a Task Environment?

A task environment defines the problem space in which an intelligent agent operates. It specifies the goals of the agent, what it can perceive, and what actions it can perform.

Before designing an agent, we must clearly describe the environment in which it will act. This prevents ambiguity and helps ensure rational behavior.

A task environment answers the question: “What exactly is the agent supposed to do?”

2. What Is PEAS?

PEAS is a framework used to specify a task environment in Artificial Intelligence. It stands for:

  • P – Performance Measure
  • E – Environment
  • A – Actuators
  • S – Sensors

The PEAS framework ensures that all critical aspects of the agent’s interaction with the world are considered.

3. Performance Measure (P)

The performance measure defines how the success of the agent is evaluated. It reflects the designer’s objectives and determines what behavior is considered rational.

  • Must be objective and measurable
  • Should align with real-world goals
  • May involve multiple criteria
Example: For a self-driving car: safety, travel time, fuel efficiency, and comfort.

4. Environment (E)

The environment is everything external to the agent that it interacts with. It includes physical, digital, social, and economic factors.

  • Physical world (roads, buildings, obstacles)
  • Other agents (humans, robots, competitors)
  • Rules and constraints

The complexity of the environment heavily influences the design of the agent.

5. Actuators (A)

Actuators are the mechanisms through which an agent acts upon the environment. They convert decisions into actions.

  • Motors, wheels, robotic arms
  • Speech or display outputs
  • Digital actions (API calls, database updates)
Actuators define what the agent is capable of doing.

6. Sensors (S)

Sensors allow the agent to perceive its environment. They provide the raw information required for decision-making.

  • Cameras, microphones, GPS
  • Keyboard and mouse input
  • System logs and data streams

An agent’s intelligence is limited by the quality and scope of its sensors.

7. Complete PEAS Example: Autonomous Taxi

PEAS Component Description
Performance Measure Safety, legality, passenger satisfaction, profit
Environment Roads, traffic, pedestrians, weather, maps
Actuators Steering, accelerator, brake, horn, display
Sensors Cameras, lidar, radar, GPS, speedometer

8. Why PEAS Is Important

  • Clarifies agent objectives
  • Prevents incomplete problem definitions
  • Guides architecture and algorithm selection
  • Improves evaluation and testing
A well-defined PEAS description is the foundation of any intelligent agent design.

9. PEAS and Rationality

Rationality is defined with respect to the performance measure in a given task environment. Without PEAS, rational behavior cannot be meaningfully evaluated.

An agent may appear irrational if the performance measure is poorly designed.

10. Summary

The PEAS framework provides a systematic way to specify task environments by clearly defining goals, surroundings, actions, and perceptions.

PEAS transforms vague problems into precise, solvable AI tasks.
