PART III: Problem Solving by Search

Problem Formulation and Problem Definition

1. Introduction to Problem Solving by Search

One of the fundamental capabilities of an intelligent agent is the ability to solve problems. In Artificial Intelligence, problem solving is often modeled as a process of searching through a space of possible states to find a sequence of actions that leads from an initial situation to a desired goal.

This approach is known as problem solving by search. Before an agent can apply any search algorithm, it must first precisely understand and represent the problem it is trying to solve. This is achieved through problem formulation and problem definition.

2. Problem Formulation

Problem formulation is the process of deciding how to represent a real-world problem in a way that an AI agent can reason about and solve using search techniques.

The real world is complex and continuous, but search algorithms operate on well-defined, abstract models. Problem formulation bridges this gap by selecting the relevant aspects of the world and ignoring unnecessary details.

2.1 Purpose of Problem Formulation

  • To convert an informal problem into a formal structure
  • To define what states and actions are relevant
  • To make the problem solvable using search algorithms
  • To balance realism and computational feasibility
Important Insight: A poorly formulated problem can be extremely hard or impossible to solve, even if a solution exists.

3. Problem Definition

Once the problem is formulated conceptually, it is expressed formally as a problem definition. A problem definition specifies the exact components required by a search algorithm.

In classical AI, a problem is defined by the following five elements.

3.1 Initial State

The initial state describes the starting point of the agent in the state space.

  • Represents the current situation
  • Must be fully specified
  • Acts as the root node of the search tree

3.2 State Space

The state space is the set of all possible states that can be reached from the initial state by applying actions.

  • May be finite or infinite
  • Defines the search domain
  • Graphically represented as a graph or tree

3.3 Actions (Successor Function)

The successor function defines the available actions for each state and the resulting states after performing those actions.

  • Maps a state to possible next states
  • Defines the edges of the state space graph
  • Determines how the agent can move through the space

3.4 Goal Test

The goal test determines whether a given state satisfies the objective of the problem.

  • Used to detect successful states
  • May check a single goal or multiple goals
  • Ends the search when satisfied

3.5 Path Cost

The path cost assigns a numeric cost to a sequence of actions.

  • Measures solution quality
  • May represent time, distance, energy, or risk
  • Used to find optimal solutions

4. Example: Route Finding Problem

Initial State: Starting city (e.g., Arad)

State Space: All cities and road connections

Actions: Drive to a neighboring city

Goal Test: Reach destination city (e.g., Bucharest)

Path Cost: Total distance traveled

Once defined this way, standard search algorithms such as BFS, DFS, Uniform Cost Search, or A* can be applied.

5. Relationship Between Formulation and Definition

Problem formulation is a conceptual and modeling step, while problem definition is a precise and formal specification.

  • Formulation decides what to model
  • Definition specifies how it is represented
  • Both strongly influence search efficiency

6. Summary

Problem solving by search relies on accurately representing problems in a structured manner. Problem formulation abstracts the real world into a solvable model, while problem definition provides the exact components required by search algorithms.

A well-defined problem is the foundation upon which all search strategies operate.
