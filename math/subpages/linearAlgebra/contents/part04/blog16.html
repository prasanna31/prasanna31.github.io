Linear Transformations — Complete Formulae and Explanation

This guide explains linear transformations from the ground up, with every key formula paired with intuitive explanations. After reading this, you'll understand the concept algebraically, geometrically, and practically.

1. Definition

A linear transformation is a function T: V → W between vector spaces V and W that preserves vector addition and scalar multiplication.

T(u + v) = T(u) + T(v), ∀ u,v ∈ V
T(c * u) = c * T(u), ∀ u ∈ V, c ∈ ℝ

Explanation:

  • Linearity ensures that straight lines remain straight and proportions are preserved.
  • The origin is always mapped to the origin: T(0) = 0.
  • All linear combinations are preserved: T(c₁v₁ + ... + cₖvₖ) = c₁T(v₁) + ... + cₖT(vₖ).

2. Matrix Representation

Every linear transformation in finite dimensions can be represented by a matrix. If T: ℝⁿ → ℝᵐ, there exists a matrix A ∈ ℝᵐˣⁿ such that:

T(v) = A * v

Explanation:

  • The columns of A are the images of the basis vectors of ℝⁿ.
  • Matrix multiplication automatically preserves linearity.
T: ℝ² → ℝ², T(x, y) = (2x + y, x - y)
Matrix: A = [[2, 1], [1, -1]]
For v = [3,4]: T(v) = A*v = [10, -1]

3. Common Linear Transformations and Formulae

3.1 Scaling

T(x, y) = (αx, βy)

Explanation: Stretches/shrinks the vector along each axis independently. Lines through the origin remain lines.

3.2 Rotation (2D)

T([x, y]) = [x cosθ - y sinθ, x sinθ + y cosθ]

Explanation: Rotates every vector around the origin by angle θ. Lengths and angles are preserved (orthogonal transformation).

3.3 Reflection

Reflection about x-axis: T(x, y) = (x, -y)
Reflection about y-axis: T(x, y) = (-x, y)
Reflection about y=x: T(x, y) = (y, x)

Explanation: Flips vectors over a line or plane. Origin remains fixed, and linearity is preserved.

3.4 Shear

Shear along x-axis: T(x, y) = (x + ky, y)
Shear along y-axis: T(x, y) = (x, y + kx)

Explanation: Slants shapes. Squares become parallelograms, but straight lines and origin are preserved.

3.5 Projection

Projection onto x-axis: T(x, y) = (x, 0)
Projection onto y-axis: T(x, y) = (0, y)

Explanation: Collapses vectors onto a subspace. Lines along the subspace remain unchanged, lines perpendicular collapse.

4. Kernel and Image

4.1 Kernel (Null Space)

Ker(T) = {v ∈ V | T(v) = 0}

Explanation: Set of vectors that get mapped to zero. Geometrically, vectors “collapse” to the origin under T.

4.2 Image (Range)

Im(T) = {w ∈ W | ∃ v ∈ V such that T(v) = w}

Explanation: All possible outputs of T. Geometrically, it is the subspace of W that T “covers”.

5. Composition and Inverse

5.1 Composition

If T₁: V → W and T₂: W → U, then T₂ ∘ T₁: V → U is linear

Explanation: Applying two linear transformations one after the other still preserves linearity.

5.2 Inverse

If T is invertible, then T⁻¹(T(v)) = v

Explanation: Reverses the effect of T. Exists if the matrix representing T is non-singular (det(A) ≠ 0).

6. Linear Combination Preservation

T(c₁v₁ + c₂v₂ + ... + cₖvₖ) = c₁T(v₁) + c₂T(v₂) + ... + cₖT(vₖ)

Explanation: Any linear combination of vectors before transformation is the same as the combination of their transformed vectors. This is the key property that defines linearity.

7. Geometrical Interpretation

  • Lines and planes through the origin remain lines and planes.
  • Scaling stretches/shrinks space along axes.
  • Rotation moves all vectors around origin by θ.
  • Reflection flips vectors across axes or planes.
  • Shear slants shapes but preserves straight lines.
  • Projection collapses vectors onto a lower-dimensional subspace.

8. Higher-Dimensional Transformations

T(v) = A*v, v ∈ ℝⁿ, A ∈ ℝⁿˣⁿ

Examples:

Scaling: T(x, y, z) = (αx, βy, γz)
Projection onto xy-plane: T(x, y, z) = (x, y, 0)
Rotation about z-axis: T([x, y, z]) = [x cosθ - y sinθ, x sinθ + y cosθ, z]

9. Applications

  • Computer Graphics: rotation, scaling, reflection, shear, projection
  • Machine Learning: PCA, embeddings, feature transformations
  • Physics: rotation matrices, stress-strain mapping
  • Engineering: signal processing, control systems
  • Economics: linear approximations and input-output models
