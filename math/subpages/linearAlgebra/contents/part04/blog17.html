Kernel and Image — What Is Lost and What Survives

Linear transformations are like magical lenses that reshape space. Some directions in the domain disappear entirely when viewed through the transformation, while others survive and form the resulting image. Understanding kernel and image allows us to see exactly what is lost and what survives, both algebraically and geometrically. This page explains these concepts in depth, step by step.

1. The Kernel (Null Space) — The Lost Directions

The kernel of a linear transformation T: V → W is the set of all vectors in the domain that get mapped to the zero vector in the codomain:

Ker(T) = { v ∈ V | T(v) = 0 }

Intuitive Explanation:

  • Vectors in the kernel are “invisible” to the transformation — no matter how long or big they are, they collapse to zero.
  • Geometrically, the kernel is a subspace of the domain that disappears completely in the output.
  • The number of independent directions in the kernel is called the nullity of T. These are exactly the directions that get “lost”.
  • Every vector in the domain can be decomposed into a sum of a kernel component (lost) and a component that survives (contributes to the image).
Example 1:
T: ℝ² → ℝ², T(x, y) = (x - y, x - y)
Solve T(x, y) = (0,0) → x - y = 0 → y = x
Ker(T) = { t(1,1) | t ∈ ℝ }

All vectors along the line y = x collapse to zero. These directions are “lost” in T.

Example 2 (3D):
T: ℝ³ → ℝ², T(x, y, z) = (x + y, y + z)
Solve T(x, y, z) = (0,0): x + y = 0, y + z = 0 → x = -y, z = -y
Ker(T) = { t(-1,1,-1) | t ∈ ℝ }

All vectors along the line in direction (-1,1,-1) vanish under T. This shows how in higher dimensions, the kernel can be a line, plane, or higher-dimensional subspace.

2. The Image (Range) — The Surviving Directions

The image of a linear transformation T is the set of all vectors in the codomain that are actually produced as outputs:

Im(T) = { w ∈ W | ∃ v ∈ V such that T(v) = w }

Intuitive Explanation:

  • The image contains all directions in W that “survive” the transformation.
  • Geometrically, it is a subspace of the codomain formed by the vectors that T can reach.
  • The dimension of the image is called the rank of T. These are exactly the independent directions that survive.
  • Every vector in the codomain not in the image is “unreachable” by T.
Example 1:
T: ℝ² → ℝ², T(x, y) = (x - y, x - y)
The output is always along the line y = x:
Im(T) = { t(1,1) | t ∈ ℝ }

Explanation: Every output vector lies on the line y = x in ℝ². This line is the “surviving” part of T.

Example 2:
T: ℝ³ → ℝ², T(x, y, z) = (x + y, y + z)
By varying x, y, z, all points in ℝ² can be reached:
Im(T) = ℝ²

Here, everything in the codomain survives. The kernel accounts for the “lost” direction, and the rest fills the codomain.

3. Kernel vs Image — The Lost and Survived Components

Every vector in the domain can be thought of as having two components:

  • Kernel Component: part that collapses to zero → lost
  • Image Component: part that contributes to output → survives

Mathematically, for any v ∈ V:

v = v_kernel + v_survive, where T(v_kernel) = 0 and T(v_survive) ∈ Im(T)

Geometrically:

  • v_kernel lies in the subspace that gets mapped to the origin.
  • v_survive lies in the subspace whose image is the codomain subspace (image).

4. Rank-Nullity Theorem — Linking Lost and Survived

The rank-nullity theorem precisely relates the lost and surviving directions:

dim(V) = dim(Ker(T)) + dim(Im(T))

Explanation:

  • The total number of independent directions in the domain equals the number lost (kernel) plus the number that survive (image).
  • Nullity = dim(Ker(T)) → lost directions
  • Rank = dim(Im(T)) → surviving directions
  • This theorem ensures we can always account for “everything” in the domain — either lost or survived.
T: ℝ³ → ℝ², T(x, y, z) = (x + y, y + z)
Ker(T) = { t(-1,1,-1) | t ∈ ℝ } → dim(Ker(T)) = 1
Im(T) = ℝ² → dim(Im(T)) = 2
Check: dim(V) = 3 = 1 (lost) + 2 (survived) ✅

5. Geometrical Visualization

Imagine a 3D cube in the domain:

  • Vectors along the kernel collapse → cube is “flattened” along these directions.
  • Vectors along other directions survive → cube projects onto the image subspace in codomain.
  • Projection, rotation, and shear are good ways to see this: kernel = collapsed direction, image = surviving plane/line.
Kernel = “lost directions” → collapse to zero.
Image = “surviving directions” → fill the codomain subspace.

6. Applications of Lost vs Survived

  • Data Compression & PCA: Kernel directions = zero-variance → lost. Image = retained components → survive.
  • Linear Systems: Kernel = free variables in homogeneous solution → lost. Image = constraints that survive.
  • Computer Graphics: Projection onto a screen → kernel = depth direction lost. Image = visible screen coordinates survive.

7. Summary and Key Formulas

Ker(T) = { v ∈ V | T(v) = 0 }   (Lost directions)
Im(T) = { w ∈ W | ∃ v ∈ V: T(v) = w }   (Surviving directions)
v = v_kernel + v_survive, T(v_kernel) = 0, T(v_survive) ∈ Im(T)
dim(V) = dim(Ker(T)) + dim(Im(T))   (Rank-Nullity Theorem)

By understanding kernel as the “lost” part of a transformation and image as the “surviving” part, one can fully grasp the behavior of linear transformations in any dimension, both algebraically and geometrically.
