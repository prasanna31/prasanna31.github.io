From Linear Maps to Matrices

In linear algebra, abstract linear maps (or linear transformations) are functions between vector spaces that preserve vector addition and scalar multiplication. To compute with them, we often represent these maps as matrices relative to chosen bases. This bridge allows us to move from theory to concrete computation.

1. Linear Map Definition

Let V and W be vector spaces over the same field F. A function T: V → W is called a linear map if, for all u,v ∈ V and c ∈ F:

T(u + v) = T(u) + T(v),
T(c * u) = c * T(u)

Intuition: Linear maps preserve the "structure" of the vector space.

2. Representing Linear Maps with Matrices

To compute with a linear map, we select a basis for V (domain) and W (codomain). Let:

  • B = {v₁, ..., vₙ} be a basis for V
  • C = {w₁, ..., wₘ} be a basis for W

Then, for each basis vector vᵢ ∈ B, express T(vᵢ) as a linear combination of C:

T(vᵢ) = a₁ᵢ w₁ + a₂ᵢ w₂ + ... + aₘᵢ wₘ

The coefficients form the i-th column of the matrix representing T:

[T]_{C←B} = \begin{bmatrix} a₁₁ & a₁₂ & ... & a₁ₙ \\ a₂₁ & a₂₂ & ... & a₂ₙ \\ ... & ... & ... & ... \\ aₘ₁ & aₘ₂ & ... & aₘₙ \end{bmatrix}

Where [T]_{C←B} is the matrix representing T from basis B of V to basis C of W.

3. Computing Coordinates

If v ∈ V has coordinates [v]_B relative to basis B, then the coordinates of T(v) relative to basis C are:

[T(v)]_C = [T]_{C←B} [v]_B

This formula is the computational bridge between abstract linear maps and matrix arithmetic.

4. Example: Linear Map in ℝ² → ℝ²

Let T: ℝ² → ℝ² be defined by T(x,y) = (2x + y, x - y)
Choose standard bases B = C = {e₁=(1,0), e₂=(0,1)}
  • T(e₁) = T(1,0) = (2,1) → first column of matrix
  • T(e₂) = T(0,1) = (1,-1) → second column of matrix
Hence, the matrix representation is:
[T] = \begin{bmatrix} 2 & 1 \\ 1 & -1 \end{bmatrix}

Given v = (3,4), coordinates [v]_B = [3,4]ᵀ. Then:

[T(v)]_C = [T] [v]_B = \begin{bmatrix}2 & 1 \\ 1 & -1\end{bmatrix} \begin{bmatrix}3 \\ 4\end{bmatrix} = \begin{bmatrix}10 \\ -1\end{bmatrix}

5. Changing Bases

If we choose different bases B' for V and C' for W, the matrix representation changes. Let P be the change of basis matrix from B' to B, and Q from C' to C. Then the new representation is:

[T]_{C'←B'} = Q⁻¹ [T]_{C←B} P

Intuition: the linear map itself doesn’t change; only its coordinate representation does. This is critical in diagonalization, similarity transformations, and simplifying linear systems.

6. Geometric Intuition

  • Think of vectors in V as arrows in space and T as a “transformation” that stretches, rotates, or shears them.
  • Choosing a basis is like choosing a coordinate grid; the same transformation looks different in different grids.
  • The matrix encodes exactly how basis vectors are mapped, which allows us to compute the effect on any vector.

7. Steps to Represent a Linear Map as a Matrix

  1. Choose a basis B for the domain V and a basis C for the codomain W.
  2. Apply T to each vector in B: T(v₁), ..., T(vₙ).
  3. Express each T(vᵢ) as a linear combination of vectors in C.
  4. Form the matrix [T]_{C←B} by placing these coordinate vectors as columns.
  5. Use [T(v)]_C = [T]_{C←B} [v]_B for computations.

8. Applications

  • Solving systems of linear equations: representing linear transformations as matrices allows computation via Gaussian elimination.
  • Diagonalization and Jordan form: changing basis to simplify matrix representation.
  • Computer graphics: representing rotations, scalings, and transformations as matrices for rendering.
  • Data science: transforming coordinates in PCA, linear regression, and other algorithms.
  • Control theory: representing state-space systems as matrices.

9. Summary Formulas

T(v) linear map: V → W
[T]_{C←B} = [T(v₁)_C ... T(vₙ)_C]
[T(v)]_C = [T]_{C←B} [v]_B
[T]_{C'←B'} = Q⁻¹ [T]_{C←B} P

By representing linear maps as matrices, we convert abstract operations into concrete, computable forms while preserving the structure of the transformation.
